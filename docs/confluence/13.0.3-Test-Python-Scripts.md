## Gather a VMS log bundle

Before you handoff to the customer, gather a log bundle.

```
sudo tar cvfz /userdata/$(hostname)-vms_logs.tgz /vast/vman/vms/log /vast/data/vms-bringup.log /var/log/messages
```

SCP this file to your laptop.

Edit HOSTNAME below to reflect the proper filename. You likely will not know this value until you are ready to perform this step.

```
scp -o "StrictHostKeyChecking=no" -o "UserKnownHostsFile /dev/null" vastdata@192.168.2.2:/userdata/c-128-2-vms_logs.tgz .
```

### CLI

Connect to VMS GUI and create your VIP(s):

```
# create VIP via CLI
vcli
# create a vip pool and note where the IPs end up. That matters for next test.
vippool create --start-ip DATAIP1 --end-ip DATAIP2 --subnet-cidr DCIDR --gw-ip DGW
vip list
```

Verify that VIPs are setup and listening:

```
showmount -e DATAVIPNOTONTHISCNODE
# make sure the above command shows you one export.  If you get an error, double check the VIP Pool you created.
```
