Before You Begin

P
Prerequisites
  • Download and install As-Built Reporter
  • Add Rack(s) to VMS
  • Add CBoxes / DBoxes / EBoxes with Rack U-Heights
  • Add Switches to VMS (customer-managed switches can be added manually)
  • Gather switch information: Name, Rack, U-Height
G
Getting Started
  • Connect directly to a Tech Port (192.168.2.2) or to VMS VIP over the network. Tech Port connections discover VMS CNode automatically.
  • Verify CNode SSH access: ssh -o "StrictHostKeyChecking=no" -o "UserKnownHostsFile /dev/null" vastdata@192.168.2.2
  • Verify Switch SSH access from CNode: ssh cumulus@<switch-ip>

Deployment Workflow

1
Connection Settings

Enter the Cluster IP and save a new profile. Default passwords are pre-populated when Autofill is enabled (see Advanced).

  • Tech Port: use 192.168.2.2.
  • Network: use the VMS VIP.
Open Reporter
2
Discovery & Switch Placement

Click Discovery to open the Switch Placement Editor. Run Discovery, assign switch U-height. Optionally, use +Switch to add customer-managed switches and assign rack U-height, then Save to Profile.

Configure
3
Run Report

Open Reporter — Under Select Operation, select:

  • Pre-Validation
  • Generate As-Built Report
  • Health Check

Click Run, then monitor progress in Output Results logging console.

Run
4
Review & Download

View PDF, Download PDF, and Download JSON buttons activate once the report completes. Browse all results on the Results page.

View Results
